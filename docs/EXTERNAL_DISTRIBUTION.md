1. 一次只给一个最小可执行动作

### 2B. English community post

We open-sourced a new skill: `small-step-collaboration`

It is not about giving agents a brand new tool. It is about improving how agents collaborate with humans during real workflows.

Many failures in setup, debugging, and dashboard tasks come from pacing, not capability:

- too many steps dumped at once
- too many branches shown too early
- too many screenshots, logs, and outputs requested in one turn
- later instructions becoming stale as soon as the screen changes

`small-step-collaboration` is designed to make agents guide users one smallest useful step at a time.

Best-fit scenarios:

- Search Console / Bing Webmaster / similar webmaster tools
- Cloudflare / Vercel / GitHub / database dashboards
- setup, deployment verification, and troubleshooting

Current first-class targets:

- Codex
